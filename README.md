# roelvanderven.com

Personal website for Roel van der Ven - Product Manager, technologist, and bread baker.

Built with Hugo, self-hosted on homelab via Docker and Cloudflare Tunnel.

## How It Works

This is a static website generated by [Hugo](https://gohugo.io/), a fast static site generator written in Go.

**Architecture:**
- **Static Site Generator:** Hugo (v0.154+)
- **Content:** Markdown files in `content/` directory
- **Templates:** Custom theme in `themes/roelvanderven/`
- **Deployment:** Docker + nginx:alpine on homelab
- **Hosting:** Self-hosted on port 8095, exposed via Cloudflare Tunnel
- **Content Management:** Designed to work with Obsidian vault (see below)

**Content Types:**
- **Posts** (`/posts/`) - Long-form articles and blog posts
- **Stills** (`/stills/`) - Screenshots from movies, TV shows, and visual media
- **Photos** (`/photos/`) - Two subtypes:
  - `photography` - High-resolution photos taken by Roel
  - `cameo` - Photos of Roel collected from around the web (SEO-optimized)

## Quick Commands

### Build the site locally
```bash
hugo server
```
Visit http://localhost:1313 to preview.

### Build for production
```bash
hugo --minify
```
Generates static files in `public/` directory.

### Deploy to homelab
```bash
./publish.sh
```
This script:
1. Builds the site with `hugo --minify`
2. Copies files to homelab via scp
3. Site is immediately live at http://10.0.0.227:8095

### View Docker logs
```bash
ssh docker-host "docker logs roelvanderven-website"
```

### Restart Docker container
```bash
ssh docker-host "cd /docker/roelvanderven.com && docker compose restart"
```

## Publishing Workflow

### 1. Create New Content

#### Blog Post
Create a new markdown file in `content/posts/`:

```markdown
---
title: "My New Post"
date: 2026-01-22T15:30:00+01:00
draft: false
tags: ["product", "technology"]
image: "/images/posts/my-header-image.jpg"  # optional
mastodon_url: ""  # add after cross-posting
---

Your content here in markdown format.
```

#### Still (Screenshot)
Create a new markdown file in `content/stills/`:

```markdown
---
title: "That moment in Blade Runner 2049"
date: 2026-01-22
draft: false
tags: ["movies", "sci-fi"]
screenshot_source: "Blade Runner 2049"
screenshot_link: "https://www.imdb.com/title/tt1856101/"
image: "/images/stills/blade-runner-2049-moment.jpg"
mastodon_url: ""
---

Optional description or commentary about this moment.
```

#### Photo (Photography)
Create a new markdown file in `content/photos/`:

```markdown
---
title: "Berlin Fireworks 2024"
date: 2024-12-31
type: photography
tags: ["berlin", "night", "photography"]
image: "/images/photos/berlin-fireworks-2024.jpg"
description: "New Year's Eve fireworks over Berlin"
location: "Berlin, Germany"  # optional
mastodon_url: ""
---

Optional longer caption or story about the photo.
```

#### Photo (Cameo)
For photos of Roel from around the web:

```markdown
---
title: "Roel van der Ven at SoundCloud"
date: 2014-03-15
type: cameo
tags: ["cameo", "berlin", "soundcloud"]
image: "/images/photos/roel-van-der-ven-at-soundcloud.jpg"
alt: "Roel van der Ven at the SoundCloud office"
source_url: "https://www.instagram.com/p/xxxxx/"
source_name: "Instagram - @photographer"
---

Working at the SoundCloud office in Berlin.
```

### 2. Add Images

Place images in the appropriate directory:
- Posts: `static/images/posts/`
- Stills: `static/images/stills/`
- Photos: `static/images/photos/`

### 3. Test Locally

```bash
hugo server
```

Preview at http://localhost:1313

### 4. Publish

```bash
./publish.sh
```

Site is live immediately.

### 5. Cross-post to Mastodon (Optional)

Manually create a Mastodon post on https://social.lol/@roel, then add the URL to the post's frontmatter:

```yaml
mastodon_url: "https://social.lol/@roel/123456789"
```

Rebuild and deploy to show the "Discuss on Mastodon →" link.

## Special Features

### Exclude Posts from Feed

To hide a post from the homepage and archive listings (but keep it accessible via permalink):

```yaml
---
title: "My Hidden Post"
excludeFromFeed: true
---
```

The post will be accessible at its URL but won't appear in listings.

### Video Embeds

**YouTube:**
```markdown
{{< youtube "VIDEO_ID" >}}
```

**Vimeo:**
```markdown
{{< vimeo "VIDEO_ID" >}}
```

### Header Images

Add a featured image to any post:

```yaml
image: "/images/posts/my-header-image.jpg"
```

## Setting Up Obsidian Vault

The site is designed to work seamlessly with Obsidian for content management.

### 1. Create Obsidian Vault

Point Obsidian to open this directory as a vault:

```
~/Code/roelvandervencom-revamp
```

Or if you prefer a dedicated location:

```
~/Documents/Obsidian/Website
```

If using a separate location, symlink or copy the project there.

### 2. Install Obsidian Plugins (Recommended)

**Shellcommands Plugin:**
1. Install "Shellcommands" from Community Plugins
2. Add command: `bash ./publish.sh`
3. Assign hotkey: `Cmd+Shift+P` for Publish
4. Now you can publish directly from Obsidian!

**Git Plugin (Optional):**
- Install "Obsidian Git" for version control
- Configure to auto-commit or manual commits
- Push to GitHub for backup

### 3. Create Content Templates

In Obsidian settings → Templates, create templates for each content type:

**Template: Blog Post** (`templates/blog-post.md`)
```markdown
---
title: "{{title}}"
date: {{date}}T{{time}}+01:00
draft: false
tags: []
image: ""
mastodon_url: ""
---


```

**Template: Still** (`templates/still.md`)
```markdown
---
title: ""
date: {{date}}
draft: false
tags: ["movies"]
screenshot_source: ""
screenshot_link: ""
image: "/images/stills/"
mastodon_url: ""
---


```

### 4. Obsidian Workflow

1. Create new note in appropriate folder
2. Use template
3. Write content
4. Add images to `static/images/`
5. Press `Cmd+Shift+P` to publish (or run `./publish.sh` in terminal)
6. Site is live!

## Configuration

Main configuration is in `hugo.toml`:

```toml
baseURL = "https://roelvanderven.com/"
title = "Roel van der Ven"
theme = "roelvanderven"

[params]
  description = "Personal website of Roel van der Ven"
  author = "Roel van der Ven"
  mastodon = "https://social.lol/@roel"
  github = "https://github.com/roelven"
  linkedin = "https://linkedin.com/in/roelvanderven"
  maxHomeItems = 10  # Posts per page
```

## Deployment Details

**Server:** homelab at `/docker/roelvanderven.com`
**Port:** 8095
**Container:** nginx:alpine
**Files:** `/docker/volumes/roelvanderven.com/public`
**Cloudflare Tunnel:** Pre-configured in `/etc/cloudflared/config.yml`

To update DNS:
1. Log into Cloudflare
2. Point roelvanderven.com to the tunnel
3. Traffic flows: Internet → Cloudflare → Tunnel → localhost:8095 → nginx

## Troubleshooting

### Site not updating after publish
```bash
ssh docker-host "docker restart roelvanderven-website"
```

### Check if Hugo is working
```bash
hugo version
# Should show v0.154.5 or higher
```

### Install/Update Hugo
```bash
brew install hugo
# or
brew upgrade hugo
```

### Images not showing
- Check file paths start with `/images/`
- Verify files exist in `static/images/`
- File paths are case-sensitive

### Build errors
```bash
hugo --minify --verbose
```

This will show detailed error messages.
